## 使用者角色
1. **一般使用者 (Patient)**：記錄健康數據、查看 AI 分析、授權分享資料。
2. **健身教練 (Coach)**：獲得授權後，查看學員運動趨勢以優化訓練。
3. **營養師 (Nutritionist)**：分析生理數據提供精準飲食建議。
4. **系統管理員 (Admin)**：管理平台資源與維護 FHIR 資料格式。

## 核心 FHIR Resources
- **Patient**: 儲存使用者的基本身份資訊（姓名、性別、生日）。
- **Observation**: 儲存各種量測指標，包含：
  - Blood Pressure (血壓)
  - Body Weight (體重)
  - Heart Rate (心率)
  - BMI (身體質量指數)
  - Steps & Exercise (步數與運動紀錄)
- **Practitioner**: 平台專業醫療人員/導師資訊。

## Demo 操作流程
1. **角色登入**：點擊側邊欄「角色登入」，選擇「一般使用者」。
2. **數據輸入**：進入「資料輸入」頁面，填寫血壓、體重等數據並送出。
3. **查看 FHIR**：系統會自動將資料轉換為 FHIR 格式，可於「FHIR JSON」頁面查看標準化的 JSON 資料。
4. **AI 分析**：進入「AI 健康分析」查看根據您的數據產生的風險評估與飲食/運動建議。
5. **趨勢圖表**：於「健康趨勢」查看歷史變化的視覺化呈現。
6. **授權分享**：於「授權分享」產生 QR Code，並在「區塊鏈紀錄」中確認授權存取軌跡。

## 如何執行
本網站為純前端 SPA 設計，無需安裝任何環境：
1. 下載專案資料夾。
2. 進入 `實作內容/程式原始碼` 目錄。
3. **直接雙擊 `index.html`** 即可在瀏覽器中開啟。

## 功能說明
- **SPA 單頁應用**：流暢的頁面切換體驗，無需重新載入。
- **FHIR 引擎**：內建 JavaScript 轉換器，即時產生符合 HL7 FHIR 標準的 Observation Bundle。
- **AI 規則引擎**：模擬智慧醫護，提供包含 BMI、血壓、運動量等多維度的健康分析。
- **區塊鏈存證模擬**：展示如何利用雜湊值 (Hash) 紀錄授權行為，確保資料安全與可追溯性。
- **本地端儲存**：使用 `localStorage` 儲存您的 Demo 數據，重新整理頁面資料也不會消失。
- **響應式設計**：完美支援桌機、平板與手機顯示。
